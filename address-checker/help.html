§ 01

How it works

NZ Address Checker adds a Lookup Address action to the Customer Card and Ship-to Address card. When triggered, it opens a dialog powered by the Addy Solutions NZ address autocomplete widget.

You type the start of a New Zealand address, select from the autocomplete suggestions, and the extension populates the address fields on the record automatically — including creating or updating the Post Code and County records in Business Central if they don't already exist.

User clicks Lookup Address (or presses Alt+A)
       ↓
Address lookup dialog opens with the Addy widget
       ↓
User types a partial address — autocomplete suggestions appear
       ↓
User selects the correct address from the list
       ↓
BC populates Address, Address 2, City, Post Code, and County
       ↓
Post Code and County records are created in BC if needed
       ↓
Address is logged to the Addy Address Log

The Addy Solutions widget queries the full New Zealand address database, including urban, rural, and PO Box addresses.

§ 02

Prerequisites

Before using NZ Address Checker you need an active Addy Solutions account with an API key.

RequirementDetails
Addy Solutions account Sign up at addysolutions.com. A free tier is available for low-volume use.
API Key Available from your Addy Solutions dashboard. This is the key used by the address autocomplete widget.
NZ Country Code A Country/Region code must exist in Business Central for New Zealand (typically NZ). This is used when creating Post Code records.
Internet access The Addy Solutions widget loads from the Addy CDN. The BC server (or the user's browser, depending on your BC deployment) must be able to reach the internet.

§ 03

Initial setup

Open SHL Addy Setup by searching for it in Tell Me (the BC search). There is one setup record shared across all companies.

FieldWhat to enter
AddySolutions API Key Your API key from the Addy Solutions dashboard. This is passed to the Addy widget at startup to authorise address lookups.
AddySolutions API Endpoint The API endpoint URL from your Addy Solutions account (e.g. https://api-nz.addysolutions.com). Used for any direct server-side calls.
NZ Country Code The Country/Region code in Business Central that represents New Zealand (typically NZ). This is stamped on Post Code records created during address lookup.

DataPerCompany = false

The setup record is shared across all companies in your Business Central environment. You only need to configure it once.

Checking your NZ Country Code

To find your NZ Country/Region code in BC, search for Countries/Regions in Tell Me and look for the New Zealand entry. Note the value in the Code column and enter that same value in the NZ Country Code field on the setup page.

§ 04

Lookup on a Customer

Open a Customer Card in Business Central. The Lookup Address action appears in the &Customer action group at the top of the page.

Two ways to trigger the lookup

MethodHow
Keyboard shortcut Press Alt+A from anywhere on the Customer Card
Action button Click Lookup &Address in the Customer action group (also pinned to the promoted action bar)

Using the lookup dialog

  1. The Addy Update Address dialog opens.
  2. Click into the Address Line 1 field and start typing — for example, a street number, street name, or suburb.
  3. Autocomplete suggestions appear as you type. Select the correct address from the dropdown.
  4. The remaining fields (Address Line 2, Suburb, City, Postcode) fill in automatically.
  5. Click OK to apply the address to the Customer record, or Cancel to discard.

In addition to the standard address fields, the Customer record's Latitude and Longitude fields are not populated directly from the lookup dialog — these are stored separately and can be updated via the API if needed.

Post Code and County records are created automatically

When an address is applied, NZ Address Checker checks whether the Post Code exists in Business Central. If it does not, it creates it. If it exists but with a different city, it renames it to match the Addy data. The County (region) record is also created if it doesn't already exist.

§ 05

Lookup on a Ship-to Address

Open a Ship-to Address card (from the Customer Card → Ship-to Addresses). The Lookup Address action works identically to the Customer Card.

MethodHow
Keyboard shortcut Press Alt+A from anywhere on the Ship-to Address card
Action button Click Lookup &Address in the Processing action group

The same address field population and Post Code / County creation logic applies as on the Customer Card. Ship-to Address records do not have Latitude and Longitude fields.

§ 06

Address fields populated

When you confirm an address selection, the following fields on the Customer or Ship-to Address record are updated:

BC FieldPopulated fromNotes
Address Addy displayline The formatted first line of the address (e.g. 12 Sample Street)
Address 2 Addy address2 or boxbaglobby Secondary detail — unit/floor for urban addresses; box bag lobby name for PO Box addresses
City Addy city (falls back to mailtown if city is blank)
Post Code Addy postcode Post Code record created or renamed in BC if needed
County Addy region County record created in BC if it doesn't already exist

Address types

Addy returns different fields depending on the address type selected:

Addy TypeAddress 2 source
Urbanaddress2 (e.g. unit or floor detail)
Ruraladdress2
NonPostaladdress2
PostBox (PO Box / Private Bag)boxbaglobby (the box bag lobby name)

Latitude and Longitude (Customer only)

The Customer table has additional SHL Latitude and SHL Longitude fields added by this extension. These are visible on the Customer Card in the address area (shown as Latitude and Longitude) and are not currently populated by the lookup dialog — they are available for manual entry or integration with other processes.

§ 07

Address Log

Every address selected through the lookup dialog is saved in full to the SHL Addy Address Log. This is a read-only audit log — one row per lookup — storing the complete Addy response including all address components, the Addy record ID, LINZ ID, and a timestamp.

Open the log by searching for Addy Address Log in Tell Me.

FieldDescription
Entry No.Sequential log entry number
Retrieved DateTimeWhen the address was selected
FullThe full formatted address string returned by Addy
Display LineThe value written to the BC Address field
Post CodeThe NZ postcode
CityThe city / mail town
RegionThe region (county)
ID / LINZ ID / DPIDAddy's unique identifiers for the address record
TypeThe Addy address type (Urban, Rural, PostBox, etc.)
x / yCoordinates returned by Addy (NZTM2000 easting/northing)

Log is append-only

Entries are written when an address is confirmed and are never deleted or modified. The log provides a full history of every address lookup performed in the environment.
