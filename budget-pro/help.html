§ 01

How it works

The core idea is ownership. Every department head is given a Budget Assignment — a record that scopes them to their dimension slice of a G/L Budget. They can only see and edit entries within that scope.

The typical flow looks like this:

Finance sets up a Budget Cycle
       ↓
Finance creates Budget Assignments (one per department / owner)
       ↓
Owners export their assignment to Excel, enter numbers, import back
       ↓
Owners submit for approval
       ↓
Approver reviews and approves (or returns for revision)
       ↓
Approved entries are dimension-stamped and locked in the G/L Budget

Approved entries carry a dedicated dimension value on a configured approval dimension. This means standard BC Account Schedules and budget vs actuals reports work natively — just filter on the Approved dimension value to see only approved budget figures.

§ 02

Roles and permissions

Two permission sets ship with Budget Pro:

Permission SetWho gets itWhat they can do
Budget Pro - Admin Finance team, budget administrators Full read/write on all Budget Pro tables; create and manage cycles, assignments, formulas, payroll cost types
Budget Pro - User Department heads, budget owners Submit and view their own assignments; enter payroll budget data; read-only on setup tables

Assign via the standard BC Permission Sets page. Neither role requires SUPER.

§ 03

Initial setup

Budget Approval Setup

Open Budget Approval Setup from the Finance Manager or Accounting Manager Role Center under the Budget Pro group. There is one setup record per company.

FieldWhat to enter
Approval Dimension CodeThe dimension that Budget Pro will use to carry approval state (e.g. a dimension called STATUS or APPROVAL). Must exist in BC Dimensions before setup.
Unapproved Dim. ValueThe dimension value stamped on every new budget entry (e.g. UNAPPROVED).
Approved Dim. ValueThe dimension value stamped when an entry is approved (e.g. APPROVED).
Notification MethodHow workflow notifications are sent: Notification (in-app), Email, or Both.
Finance EmailAn email address copied on all submission and approval notification emails.

The approval dimension slot is separate from the G/L Budget's four Budget Dimensions. It is stamped into the entry's full Dimension Set, so it appears in all standard dimension reports.

Payroll Cost Types

If you will be using the Payroll Budget module, open Payroll Cost Types from the Role Center and create a row for each payroll cost category you want to budget (e.g. Base Salary, Employer NI, Pension). Link each cost type to the G/L account it should post to when payroll entries are sent to the G/L Budget.

Assignment status flow

Open → (first import) → Started
     → (submit)       → Pending Approval
     → (approve)      → Approved
     → (reopen)       → Revised
     → (re-submit)    → Pending Revision
     → (approve)      → Approved
     → (close)        → Closed  ← terminal, no further edits

The status is colour-coded in the list view:

  • Approved — Approved
  • Pending Approval / Pending Revision — Awaiting approver action
  • Revised — Returned for changes
  • Closed — Closed, no further edits

The Submission Log FactBox on the Assignment Card records every status change with a timestamp and the user who made it.

§ 06

The Excel round-trip

Export

Open the Budget Assignment Card and click Export to Excel.

The exported workbook contains:

  • One row per G/L account matching the assignment's G/L Account Filter (or all accounts if no filter is set)
  • A Prior Year Actuals column pre-populated as a reference baseline
  • A Metadata sheet with assignment details (read-only)
  • Budget Dimension values locked to the assignment's scope — no need for the user to enter dimensions manually

Import

When the owner is ready to submit their numbers, they return the Excel file and use the standard BC Import Budget from Excel flow. Budget Pro intercepts the import to:

  1. Validate accounts — entries for accounts outside the G/L Account Filter are rejected
  2. Validate dates — if Date Locked is ticked, entries outside the budget period are rejected
  3. Override dimensions — dimension values from the Excel file are ignored; the assignment's own dimension values are always stamped instead (prevents injection of incorrect dimension values)
  4. Stamp approval dimension — every imported entry receives the Unapproved Dim. Value
  5. Advance status — the assignment moves from Open → Started on the first successful import

§ 08

Approver Work Queue

Open My Budget Approvals from the Role Center (or search for Approver Work Queue).

This page shows only the Budget Assignments where you are the approver and the status is Pending Approval or Pending Revision. Use this as your daily work queue — you can Approve or Return for Revision directly from the list without opening each assignment card.

§ 09

Budget Entries

From the Budget Assignment Card, click Budget Entries to open a filtered view of the G/L Budget Entries for that assignment. You can create, edit, and delete entries directly in BC here — no Excel round-trip required for adjustments.

Entries created automatically by Budget Formulas are shown but are not editable; their Amount field is locked (a formula recalculates them whenever the source entry changes).

The Dimensions action (Shift+Ctrl+D) lets you view or edit the full dimension set on any entry.

§ 10

Spread Budget Entry

From the Budget Entries page, select a row and click Spread. This splits the entry's amount equally across multiple periods, creating one new G/L Budget Entry per period.

FieldDescription
Start DateThe date of the first generated entry
Period LengthThe interval between entries (e.g. 1M for monthly, 1Q for quarterly)
No. of PeriodsHow many entries to create

The amount is divided equally and rounded to the company's Amount Rounding Precision. The original entry is not deleted — delete it manually if you want the spread to replace it rather than add to it.

§ 11

Budget Formulas

Budget Formulas let Finance define rules that automatically derive entries on a target G/L account whenever entries are added to a source account. Common uses: overhead allocations, payroll on-costs, departmental roll-ups.

Open Budget Formulas from the Budget Assignment Card action bar or directly from the Role Center.

Formula fields

FieldDescription
Budget NameThe G/L Budget this formula applies to
Source AccountThe account that triggers the formula
OperatorFixed Price, +, −, ×, or ÷
ValueThe operand (used with Fixed Price, Multiply, Add, Subtract, Divide)
Target AccountThe account the derived entry is written to
Dimension Code / ValueOptional — if set, the formula only fires when the source entry has this dimension value

How operators work

OperatorDerived amount
Fixed PriceAlways the configured Value, regardless of the source amount
+Source amount + Value
−Source amount − Value
×Source amount × Value
÷Source amount ÷ Value (rounded to Amount Rounding Precision)

Behaviour

  • Derived entries are created automatically when a source entry is inserted
  • When a source entry's amount or date changes, existing derived entries are reversed (a negating entry is created) and new ones are generated with the updated values
  • When a source entry is deleted, its derived entries are reversed
  • When a formula is modified, all existing derived entries for that formula are reversed and recreated under the new formula
  • When a formula is deleted, all derived entries for that formula are reversed
  • Fixed Price entries are not recalculated when the source entry changes (the amount is independent of the source)
  • Derived entries cannot be edited manually; they are identified by the Created From Formula flag

§ 12

Payroll Budget module

The Payroll Budget module lets budget owners enter detailed payroll costs by employee and cost type, then roll them up into the G/L Budget as aggregated entries.

Payroll Cost Types

Set up under Payroll Cost Types (Role Center or search). Each cost type represents a category of payroll cost (e.g. Base Salary, Employer NI, Pension) and links to the G/L account entries are posted to when rolled up.

Payroll Budget Worksheet

From the Budget Assignment Card, click Payroll Budget to open the worksheet for that assignment.

Enter one row per payroll item:

FieldDescription
Employee No.The employee this entry relates to
Employee NameFilled automatically from the employee record
Payroll Cost TypeThe cost category (links to the G/L account)
DateThe period date
AmountThe budgeted amount

Roll-up By

On the Budget Assignment, the Payroll Roll-up By field controls how payroll entries are aggregated when sent to the G/L Budget:

OptionHow entries are created
Cost TypeOne G/L Budget Entry per Cost Type + Date combination, summing across all employees
EmployeeOne G/L Budget Entry per Employee + Cost Type + Date combination

Sending to the G/L Budget

Click Send to G/L Budget on the worksheet toolbar. Budget Pro aggregates the payroll entries according to the Roll-up By setting and inserts G/L Budget Entries. After sending, you are offered the option to clear the payroll worksheet entries. If you decline, the worksheet entries remain and can be sent again (useful if you want to adjust and re-send).

If Budget Formulas are configured on the payroll G/L accounts, those formulas will fire automatically when the G/L Budget Entries are created, generating any downstream derived entries (e.g. an employer NI percentage formula).

Tips

Tips & tricks

  • Approved-only reporting: In Account Schedules or the standard BC Budget page, filter on your Approved Dim. Value to report on approved budget figures only. Entries that have not yet been approved carry the Unapproved Dim. Value and will be excluded.
  • Adjusting an approved budget: Use Reopen on the Assignment Card. This removes the entry locks and reverts entries to the Unapproved Dim. Value. The owner can then modify entries and re-submit.
  • The G/L Account Filter: Use this to restrict a budget owner to a subset of accounts (e.g. only expense accounts 6000–6999). Enter a standard BC filter expression. Entries outside the filter are rejected on import.
  • Date Locked: Tick this on the assignment and set the Budget Start/End Date to prevent owners from importing entries for dates outside the agreed budget period.
